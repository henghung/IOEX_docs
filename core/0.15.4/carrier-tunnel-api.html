IOEX Carrier Tunnel APIs¶

Data types¶

IOEXTunnelStatus¶

enum IOEXTunnelStatus¶

Status of a tunnel.

Values:

IOEXTunnelStatus_None¶

The tunnel is not setup or status unknown.

IOEXTunnelStatus_Pending¶

The tunnel is waiting for user’s approval.

IOEXTunnelStatus_Waiting¶

The tunnel is waiting for response from the other peer.

IOEXTunnelStatus_Running¶

The tunnel is running.

IOEXTunnelStatus_Finished¶

The tunnel is finished or canceled.

IOEXTunnelInfo¶

struct IOEXTunnelInfo¶

Tunnel information is used to track opened tunnel with friends.

Each tunnel is uniquely identified by randomly generated tunnel_id, which is between 1~65535. The id is generated by each side so the friend is very likely using different id (remote id). During the transmission, we need to include the remote id in the packet.

Public Members

uint16_t tunnel_id¶

The ID of the tunnel.

uint16_t remote_id¶

The ID to identify this tunnel in the remote side.

char friend_id[IOEX_MAX_ID_LEN + 1]¶

The ID of the friend who setup the tunnel with us.

IOEXTunnelStatus status¶

Current status of this tunnel.

Tunnel Functions¶

IOEX_tunnel_request¶

int IOEX_tunnel_request(IOEXCarrier *carrier, uint16_t *tunnel_id, const char *friend_id, const char *password)¶

An application-defined function that send tunnel request to a friend.

Return

0 if the request successfully sent. Otherwise, return -1, and a specific error code can be retrieved by calling IOEX_get_error().

Parameters

  • carrier: [in] A handle to the Carrier node instance.

  • tunnel_id: [out] A randomly generated ID to uniquely identify this tunnel.

  • friend_id: [in] ID of the friend we want to setup tunnel with.

  • password: [in] A pre-shared string for the friend to authenticate us.
