Data types¶

Bootstrap¶

struct BootstrapNode¶

Bootstrap defines a couple of perperities to provide for Carrier nodes to connect with.

The bootstrap nodes help Carrier nodes be connected to the others with more higher possibilities.

Public Members

const char *ipv4¶

The ip address supported with ipv4 protocol.

const char *ipv6¶

The ip address supported with ipv6 protocol.

const char *port¶

The ip port.

The default value is 33445.

const char *public_key¶

The unique public key to provide for Carrier nodes, terminated by null-string.

The length of public key is about 45 bytes.

IOEXOptions¶

struct IOEXOptions¶

IOEXOptions defines several settings that control the way the Carrier node connects to others.

Remark

Default values are not defined for persistent_location of Carrier- Options, so application should be set persistent_location clearly. If the IOEXOptions structure is defined as a static variable, initialization (in compliant compilers) sets all values to 0 (NULL for pointers).

Public Members

const char *persistent_location¶

The application defined persistent data location.

The location must be set.

const char *storage_location¶

The application defined storage location.

The location must be set.

const char *key_file_location¶

The application defined key file location.

The location must be set.

bool udp_enabled¶

The option to decide to use udp transport or not.

Setting this option to false will force Carrier node to use TCP only, which will potentially slow down the message to run through.

bool local_discovery_enabled¶

The option to decide to use LAN discovery or not.

Setting this option to false will prevent carriers from discovery friends in the local network, which will also prevent carriers from using friends as bootstrap nodes.

size_t bootstraps_size¶

The total number of bootstrap nodes to connect.

There must have at least one bootstrap node for the very first time to create carrier instance.

BootstrapNode *bootstraps¶

The array of bootstrap nodes.

Values:

The size of plane data is derived from width and height as documented below.

  • u: [in] U chroma plane. Size = MAX(width/2, abs(ustride)) * (height/2).

  • v: [in] V chroma plane. Size = MAX(width/2, abs(vstride)) * (height/2).
