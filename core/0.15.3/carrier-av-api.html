Common Control Functions¶

IOEX_av_call¶

int IOEX_av_call(IOEXCarrier *carrier, const char *friend_id, uint32_t audio_bitrate, uint32_t video_bitrate)¶

An application-defined function that send a av request to friend.

Return

0 if the request successfully sent. Otherwise, return -1, and a specific error code can be retrieved by calling IOEX_get_error().

Parameters

  • carrier: [in] A handle to the Carrier node instance.

  • friend_id: [in] ID of the friend.

  • audio_bitrate: [in] Our audio transmitting bitrate in kbps. Set 0 to disable audio.

  • video_bitrate: [in] Our video transmitting bitrate in kbps. Set 0 to disable video.

An application-defined function that set audio bit rate (in kbps) in an AV call.

Usually numbers are 64, 96, 128 and 192, and the valid range is 6 ~ 510. Set 0 will disable audio transmitting.

Parameters

  • carrier: [in] A handle to the Carrier node instance.

  • friend_id: [in] ID of the friend.

  • bitrate: [in] Bit rate of the audio. Set 0 will disable audio transmitting.

IOEX_av_send_audio¶

int IOEX_av_send_audio(IOEXCarrier *carrier, const char *friend_id, const int16_t *pcm, size_t sample_count, uint8_t channels, uint32_t sampling_rate)¶

An application-defined function that sends an audio frame to a friend.

The expected format of the PCM data is: [s1c1][s1c2][…][s2c1][s2c2][…]… Meaning: sample 1 for channel 1, sample 1 for channel 2, … For mono audio, this has no meaning, every sample is subsequent. For stereo, this means the expected format is LRLRLR… with samples for left and right alternating.

Return

0 if the request successfully sent. Otherwise, return -1, and a specific error code can be retrieved by calling IOEX_get_error().

Parameters

  • carrier: [in] A handle to the Carrier node instance.

  • friend_id: [in] ID of the friend.

  • pcm: [in] An array of audio samples. The size of this array must be sample_count * channels.

  • sample_count: [in] Number of samples in this frame. Valid numbers here are ((sample rate) * (audio length) / 1000), where audio length can be 2.5, 5, 10, 20, 40 or 60 millseconds.

  • channels: [in] Number of audio channels. Supported values are 1 and 2.

  • sampling_rate: [in] Audio sampling rate used in this frame. Valid sampling rates are 8000, 12000, 16000, 24000, or 48000.

IOEX_av_send_video¶

int IOEX_av_send_video(IOEXCarrier *carrier, const char *friend_id, uint16_t width, uint16_t height, const uint8_t *y, const uint8_t *u, const uint8_t *v)¶

An application-defined function that sends a video frame to a friend.

Return

0 if the request successfully sent. Otherwise, return -1, and a specific error code can be retrieved by calling IOEX_get_error().

Parameters

  • carrier: [in] A handle to the Carrier node instance.

  • friend_id: [in] ID of the friend.

  • width: [in] Width of the frame in pixels.

  • height: [in] Height of the frame in pixels.

  • y: [in] Y (Luminance) plane data. Size must be height * width.

  • u: [in] U (Chroma) plane data. Size must be (height/2) * (width/2).

  • v: [in] V (Chroma) plane data. Size must be (height/2) * (width/2).
