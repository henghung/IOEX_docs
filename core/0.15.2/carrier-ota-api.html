HTTP Functions¶

IOEX_http_conf¶

int IOEX_http_conf(IOEXCarrier *carrier, const char *url)¶

An application-defined function that set the url to IOEX server.

By default, the URL is set to IOEX_CARRIER_API_URL

Return

0 if the url is set successfully. Otherwise, return -1, and a specific error code can be retrieved by calling IOEX_get_error().

Parameters

  • carrier: [in] A handle to the Carrier node instance.

  • url: [in] The url of the IOEX server.

IOEX_http_get_data_count¶

int IOEX_http_get_data_count(IOEXCarrier *carrier, const char *file_id, const char *partname, int *count, int *status)¶

An application-defined function that retrieve the total count of the specific fileid and data file name set stored in the IOEX network.

We need to determine if the desired amount of data is spread in the network by using the API. The count is retrieved from IOEX server directly, rather than from IOEX network.

Return

0 if the count is retrieved successfully. Otherwise, return -1, and a specific error code can be retrieved by calling IOEX_get_error().

Parameters

  • carrier: [in] A handle to the Carrier node instance.

  • file_id: [in] Id of the file.

  • partname: [in] Name of a data part.

  • count: [out] Total count of the id/name pair stored in the network.

  • status: [out] The status return from the server.

Upload & Download¶

IOEX_upload_with_id¶

int IOEX_upload_with_id(IOEXCarrier *carrier, size_t file_count, size_t target_count, const char *fullpath, const char *fileid)¶

An application-defined function that upload and spread a local file to IOEX network.

This function will 1. compress the file, 2. split the file into pieces, 3. generate ID for each data pieces and post the ID list to IOEX server, 4. spread the data to the network by using data put. This function returns after the final data put call, regardless to actually data transmission progress.

Return

0 if the file is uploaded successfully. Otherwise, return -1, and a specific error code can be retrieved by calling IOEX_get_error().

Parameters

  • carrier: [in] A handle to the Carrier node instance.

  • file_count: [in] Number of file blocks.

  • target_count: [in] Desired number of copies of each file block.

  • fullpath: [in] The path of the local file.

  • fileid: [in] The file id for the uploaded file.

IOEX_download¶

int IOEX_download(IOEXCarrier *carrier, const char *fileid, const char *filename, const char *filepath)¶

An application-defined function that download a file from IOEX network by ID and store it locally.

This function will 1. get id list from IOEX server, 2. download each data pieces by calling data lookup. This function returns after the final data lookup call, regardless to actually data transmission progress.

Return

0 if the file is download successfully. Otherwise, return -1, and a specific error code can be retrieved by calling IOEX_get_error().

Parameters

  • carrier: [in] A handle to the Carrier node instance.

  • fileid: [in] The id of the file to be downloaded.

  • filename: [in] The name of the file used to store the downloaded file locally.

  • filepath: [in] Path to store the downloaded file.
