IOEX_GROUP_MSG_ACTION_TYPE¶

enum IOEX_GROUP_MSG_ACTION_TYPE¶

Carrier group action message type.

Values:

enumerator IOEX_GROUP_MSG_ACTION_DELETE_GROUP¶

To delete a group while using action message.

If delete a group successfully, the action message sender will receive a peer state chaned callback response with type IOEX_GROUP_PEER_ON_DELETE. Otherwise the sender will receive kick out error response with type IOEX_GROUP_MSG_ACTION_DELETE_GROUP_ERR.

enumerator IOEX_GROUP_MSG_ACTION_DELETE_GROUP_ERR¶

To delete a group unsuccessfully while using action message.

IOEX_group_chat_send_audio¶

int IOEX_group_chat_send_audio(IOEXCarrier *carrier, uint32_t group_number, const int16_t *pcm, unsigned int samples, uint8_t channels, uint32_t sampling_rate)¶

An application-defined function that sends a audio frame to a group.

The expected format of the PCM data is: [s1c1][s1c2][…][s2c1][s2c2][…]… Meaning: sample 1 for channel 1, sample 1 for channel 2, … For mono audio, this has no meaning, every sample is subsequent. For stereo, this means the expected format is LRLRLR… with samples for left and right alternating.

Parameters

  • carrier – [in] A handle to the Carrier node instance.

  • group_number – [in] Number of the group.

  • pcm – [in] An array of audio samples. The size of this array must be sample_count * channels.

  • samples – [in] Number of samples in this frame. Valid numbers here are ((sample rate) * (audio length) / 1000), where audio length can be 2.5, 5, 10, 20, 40 or 60 millseconds.

  • channels – [in] Number of audio channels. Supported values are 1 and 2.

  • sampling_rate – [in] Audio sampling rate used in this frame. Valid sampling rates are 8000, 12000, 16000, 24000, or 48000.

Returns

0 if the request successfully sent. Otherwise, less than 0, and a specific error code can be retrieved by calling IOEX_get_error().
