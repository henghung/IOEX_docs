VPN Functions¶

IOEX_vpn_request¶

int IOEX_vpn_request(IOEXCarrier *carrier, const char *friend_id, const uint8_t *extra, size_t length)¶

An application-defined function that send VPN request to a friend.

Parameters

  • carrier – [in] A handle to the Carrier node instance.

  • friend_id – [in] ID of the friend we want to setup VPN with.

  • extra – [in] Extra data you want to send to the friend. Useful in authentication process.

  • length – [in] Size of the extra data in bytes.

Returns

0 if the request successfully sent. Otherwise, return -1, and a specific error code can be retrieved by calling IOEX_get_error().

IOEX_vpn_handshake¶

int IOEX_vpn_handshake(IOEXCarrier *carrier, const char *friend_id, const uint8_t *extra, size_t length)¶

An application-defined function that send VPN handshake to a friend.

Handshake is mean to authenticate each other, and exchange information. This must be called after requested VPN, but before accepted.

Parameters

  • carrier – [in] A handle to the Carrier node instance.

  • friend_id – [in] ID of the friend we want to setup VPN with.

  • extra – [in] Extra data you want to send to the friend. Useful in authentication process.

  • length – [in] Size of the extra data in bytes.

Returns

0 if the request successfully sent. Otherwise, return -1, and a specific error code can be retrieved by calling IOEX_get_error().

IOEX_vpn_accept¶

int IOEX_vpn_accept(IOEXCarrier *carrier, const char *friend_id, const uint8_t *extra, size_t length)¶

An application-defined function that accept a VPN request.

You can accept a VPN once received the request, or after handshaking process. Only the one received the request can accept it.

Parameters

  • carrier – [in] A handle to the Carrier node instance.

  • friend_id – [in] ID of the friend we want to setup VPN with.

  • extra – [in] Extra data you want to send to the friend. Usually some network configuration data.

  • length – [in] Size of the extra data in bytes.

Returns

0 if the request successfully sent. Otherwise, return -1, and a specific error code can be retrieved by calling IOEX_get_error().

IOEX_vpn_send_data_lossless¶

int IOEX_vpn_send_data_lossless(IOEXCarrier *carrier, const char *friend_id, const uint8_t *data, size_t length)¶

An application-defined function that send lossless VPN data to a friend.

In most cases, you should use IOEX_vpn_send_data_lossy instead of this lossless API.

Parameters

  • carrier – [in] A handle to the Carrier node instance.

  • friend_id – [in] ID of the friend we want to send packets to.

  • data – [in] Data payload.

  • length – [in] Size of the data in bytes.

Returns

0 if the request successfully sent. Otherwise, return -1, and a specific error code can be retrieved by calling IOEX_get_error().
