歡迎使用 Arctos Client SDK

Arctos Client SDK 提供行動裝置上的點對點視訊串流解決方案，包含裝置之間的彼此連接、點對點安全加密傳輸、視訊與音訊的即時串流功能。

SDK 亦可搭配 Arctos 的 CMS 服務，進行設備與用戶管理服務，也方便企業更容易串接自有系統。

下載

Arctos Client SDK 可以從 Gitlab Maven Repository 下載。

首先在專案中的 settings.gradle 加入 repository 資料：

dependencyResolutionManagement {
    repositories {
        google()
        mavenCentral()

        // Add the following:
        maven {
            name "Arctos Repository"
            url REPOSITORY_URL
            credentials(HttpHeaderCredentials) {
                name = REPOSITORY_USERNAME
                value = REPOSITORY_PASSWORD
            }
            authentication {
                header(HttpHeaderAuthentication)
            }
        }
    }
}

將其中的 REPOSITORY_URL、REPOSITORY_USERNAME、REPOSITORY_PASSWORD 等替換成正確的資料。

在專案的 app/build.gradle 加入引用：

implementation 'com.arctos.sdk:client-sdk:SDK_VERSION'

使用

首先請在 APP 層級的 build.gradle 中，開啟 dataBinding 功能：

dataBinding{
    enabled true
}

為了初始化 Arctos Client SDK，APP 必須讓自己的 Application 繼承 ArctosApplication。

首先在 APP 的原始碼中新增一個 MyApplication.kt，內容如下：

class MyApplication : ArctosApplication()

如果 APP 有必要複寫 onCreate 方法，務必呼叫 super.onCreate()：

class MyApplication : ArctosApplication(){
    override fun onCreate() {
        super.onCreate()  // ArctosApplication.onCreate
        // Do something...
    }
}

ArctosApplication 提供一些設定參數，供使用者調整 Arctos Client SDK 的行為。要改變參數內容，請在 MyApplication 中直接覆寫這些變數，如：

class MyApplication : ArctosApplication(){
    override val logTag: String = "MyTag"   // Change log tag for logcat
    override val logLevel: LogLevel = LogLevel.VERBOSE  // Change minimum log severity level
}

Arctos Client SDK 以 Activity 的方式提供所有的功能與操作介面。因此最簡便的使用方式就是啟動 ArctosMainActivity：

val intent = Intent(this, ArctosMainActivity::class.java)
startActivity(intent)

設計

Arctos Client SDK 的通訊介面，在行動裝置上會佔用較大版面。為方便使用者操作，建議 APP 不要使用 Toolbar，或是採用不含 Toolbar 的主題。例如在 themes.xml 中，把 APP 主題換成 NoActionBar 的版本：

<style name="Theme.App" parent="Theme.MaterialComponents.DayNight.NoActionBar"/>
